# Lessons Learned - TOEICApp（教訓ログ）
# プロジェクト固有のミスや注意点を記録するファイル

## フォーマット
```
### [日付] タイトル
- **問題**: 何が起きたか
- **原因**: なぜ起きたか
- **対策**: 次回どうするか
```

---

## ログ

### [2026-02-17] feature-proposer が空の出力を返した
- **問題**: feature-proposer エージェントが `{}` や空のJSONを返し、提案が得られなかった
- **原因**: 実行スクリプト側で出力のバリデーションがなく、空出力がそのまま結果として保存された
- **対策**: 実行スクリプトに空出力検知のバリデーションを追加する。`{}` や proposals が空の場合は警告を出す

### [2026-02-17] verifier の判定基準が不明確で混乱が生じた
- **問題**: verifier が verdict を判定する際に明確な基準がなく、approve/request_changes/reject の使い分けが曖昧だった
- **原因**: verifier.md に判定基準が明示されていなかった
- **対策**: verifier.md に具体的な判定基準を追記した（introduced_bug があれば reject、partially_fixed が過半数なら request_changes など）

### [2026-02-19] 旧システム(coordinator.sh)と新システム(.agents/)の二重管理
- **問題**: scripts/ai-agents/coordinator.sh と .agents/ の両方にエージェント設定が存在し、どちらが正しいか混乱した
- **原因**: 新システムへの移行時に旧システムを非推奨にする明示的な記載がなかった
- **対策**: CLAUDE.md のディレクトリ構造に「scripts/ai-agents/ は非推奨」と明記。新規作業は必ず .agents/ を使用する

### [2026-02-19] Xcode プロジェクトの多重ネスト構造
- **問題**: Xcode が自動生成した `TOEICApp/TOEICApp/TOEIC/` という3段ネスト構造になっており、パス参照が複雑で管理しにくかった
- **原因**: Xcode でプロジェクト作成時にデフォルト設定のままにした結果、プロジェクト名のフォルダが二重に生成された
- **対策**: 次回 Xcode プロジェクト作成時は、プロジェクトの保存先を Git ルートと同じ場所に指定してネストを避ける。または作成直後にフラット化する（`git mv` で移動、`xcodebuild -list` で動作確認）
